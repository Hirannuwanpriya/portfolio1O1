## Overview

WeRenewSneakers is a dual-sided marketplace and e-commerce platform for sneaker
enthusiasts in Belgium — enabling users to purchase new sneakers and list their
pre-owned pairs for sale, all within a single platform. The product spans a
Laravel-powered web platform, a REST API backend, and a React Native mobile
app, with real-time synchronisation across all surfaces for orders, inventory,
and user interactions.

The team of three built the web platform and REST API collaboratively, while I
took sole ownership of the React Native mobile app — from architecture through
to App Store and Play Store deployment via Fastlane.

## The Problem

- Sneaker buyers and sellers in the Belgian market lacked a dedicated,
  trustworthy platform tailored to their buying and selling workflows.
- Generic marketplace platforms did not support the product-specific features
  sneaker enthusiasts expect: size-specific listings, condition grading,
  brand and colourway filtering, and seller reputation systems.
- Any platform serving this audience needed to work as well on mobile as on
  web — a separate, native mobile app was essential rather than a responsive
  website alone.
- Real-time inventory and order state synchronisation between web and mobile
  was a hard requirement: a purchase made on one platform had to be immediately
  reflected on the other to prevent overselling.

## My Role

Full Stack Developer and sole Mobile Developer across the 8-month engagement.
My responsibilities spanned:

- Contributing to the Laravel web platform and REST API design alongside the
  team.
- Full ownership of the React Native mobile application — architecture, UI,
  state management, payment integration, and deployment.
- Designing and implementing the REST API contract to serve both the Livewire
  web front end and the React Native mobile app from a single backend.
- Stripe payment integration on both web (Livewire) and mobile (Stripe SDK
  for React Native).
- Automated mobile deployment pipeline via Fastlane for both iOS and Android.

## Solution

### Architecture

The platform is built around a single Laravel 9 service tier that serves two
consumers: the Livewire/AlpineJS web front end and the React Native mobile app.
Laravel Sanctum handles authentication for both surfaces — web sessions for the
browser and token-based authentication for the mobile app. This single-backend
approach ensures that inventory state, order records, and user data are always
consistent regardless of which surface initiated the action.

Redux Toolkit manages client-side state in the mobile app, with Axios handling
all API communication. NativeWind was chosen as the styling layer to bring
Tailwind's utility-class system to React Native, keeping the design language
consistent between web and mobile and reducing the context-switching overhead
of maintaining two separate styling approaches.

### Web Platform

- Full marketplace and e-commerce functionality built with Laravel 9 and
  Livewire — product listings, shopping cart, checkout, and order tracking.
- Meilisearch powers the product discovery layer: instant full-text search
  with filtering by brand, size, condition, price range, and colourway.
- Laravel Nova 3 provides the admin panel for inventory management, order
  processing, seller verification, and platform moderation.
- Stripe integration handles secure payment collection for both new sneaker
  purchases and pre-owned marketplace transactions.
- Envoyer manages zero-downtime deployments to the Forge-provisioned server.

### React Native Mobile App

- Native iOS and Android app built entirely with React Native, providing a
  fast and platform-appropriate shopping experience on both operating systems.
- Full feature parity with the web platform: browse listings, search and
  filter, manage a cart, checkout via Stripe, track orders, and manage seller
  listings — all from the mobile app.
- Redux Toolkit manages all application state — cart contents, authentication,
  order history, and listing drafts — with a predictable, testable data flow.
- Stripe SDK for React Native handles the in-app payment flow with native
  payment sheet UI on both iOS and Android.
- Fastlane automates the build, signing, and submission pipeline for both
  the App Store and Google Play, reducing release overhead to a single
  command.

### Key Features

- Buy and sell marketplace — users can browse and purchase new sneakers or
  list their pre-owned pairs with condition grading, size, and pricing.
- Real-time order and inventory synchronisation between web and mobile via
  the shared REST API — purchases on either surface are immediately reflected
  across both.
- User profiles with active listings, sales history, purchase records, and
  buyer/seller interaction threads.
- Advanced search and filtering powered by Meilisearch: brand, size,
  condition, colourway, price range, and availability.
- Secure payment processing via Stripe on both web and mobile surfaces.
- Order tracking with status updates visible to both buyer and seller.
- Flare error monitoring across the Laravel backend for production visibility.

## Tech Stack Detail

Laravel Sanctum was chosen over Passport for the API authentication layer
because the mobile app's token-based flow did not require the full OAuth2
server capabilities Passport provides — Sanctum's lightweight personal access
tokens were a better fit for a mobile client authenticating against a
first-party API.

Redux Toolkit was selected over lighter state management options because the
cart, authentication, and listing flows each required optimistic updates and
coordinated async state — RTK Query provided the data-fetching and caching
layer that kept the mobile UI responsive under variable network conditions.

NativeWind brought Tailwind's utility class system to React Native, which
was a deliberate decision to reduce the visual divergence between the web
and mobile surfaces. Designers and developers working across both platforms
shared the same spacing, colour, and typography tokens without maintaining
two separate style systems.

Fastlane eliminated the manual overhead of iOS provisioning profile management
and Android signing — both release pipelines are fully automated, including
TestFlight distribution for iOS and internal track submission for Android.

## Challenges and Learnings

### Real-time inventory sync across two independent surfaces

The core technical challenge of the platform was preventing overselling: if
a user on the web and a user on the mobile app both added the same
size-specific listing to their cart and proceeded to checkout simultaneously,
only one could complete the purchase. The solution was a server-side inventory
reservation layer built into the REST API — when a checkout is initiated on
either surface, the relevant inventory unit is soft-locked for the duration
of the payment flow and released if the payment fails or times out. This
reservation state is maintained in the database and consulted by both the
Livewire checkout and the React Native Stripe payment sheet before confirming
any order.

### Building and shipping a native mobile app as the sole mobile developer

Owning the React Native app end-to-end — from the initial architecture
decision through to App Store and Play Store submission — across an 8-month
engagement required careful scope management alongside the web platform
contributions. The Fastlane pipeline was set up early in the project
specifically to remove the friction from iterative releases: with signing
and submission automated, shipping a new build for testing or review became
a routine part of the development cycle rather than a manual overhead that
competed with feature development time.
