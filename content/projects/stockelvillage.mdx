## Overview

Stockel Village is the digital platform for a major commercial district in
Brussels, covering 250+ retail shops, restaurants, personal services, a cinema
(Le Stockel), and a year-round programme of community events. The association
needed a modern, multi-language platform to replace an ageing WordPress site —
with a fully automated membership billing system layered on top to manage
business registrations and payments without ongoing manual administration.

I was the sole engineer across the entire project: migration, platform rebuild,
billing engine, multi-language CMS, and performance optimisation.

## The Problem

- The existing WordPress site was outdated and could not support the growing
  directory or the association's event and membership workflows.
- There was no automated way to manage monthly membership fees across 250+
  businesses — reminders, follow-ups, and suspensions were all handled manually.
- The platform had no multi-language support despite serving a French, Dutch,
  and English-speaking community.
- No developer handover was provided. The only access available was a WordPress
  admin username and password — no database credentials, no codebase, no
  documentation.
- The platform needed to be architected for growth, with capacity for 1,000+
  business listings as the district expanded.

## My Role

Sole engineer responsible for data migration, system architecture, frontend and
backend development, membership engine design, multi-language implementation,
and performance optimisation. The project began with a reverse-engineering phase
to extract all data and business logic from the live WordPress installation
before a single line of new code was written.

## Solution

### Phase 1 — WordPress Reverse-Engineering and Migration

With no direct database access, I worked through the entire WordPress admin
panel to map every data structure: custom post types, ACF field groups, taxonomy
hierarchies, user roles, media attachments, and plugin-generated records. Each
content type was documented and mapped to the target Laravel schema before
extraction began.

A custom migration pipeline extracted records via the WordPress REST API and
admin-accessible exports. Data quality issues — inconsistent category
assignments, duplicate entries, missing required fields — were identified and
resolved during extraction rather than carried into the new system.

### Phase 2 — Platform Architecture

Laravel service layer behind a Vue.js + TypeScript frontend. The directory was
built to support filtering by category (retail, food, services, cinema, events),
full-text search across all 250+ listings, and individual merchant profile pages
with opening hours, contact details, gallery, and event associations.

Le Stockel cinema was integrated as a dedicated section with its own content
type — separate from the standard merchant listing — to support programme
schedules and screening information managed through the CMS.

### Phase 3 — Membership Billing Engine

The membership engine runs on a scheduled Laravel job triggered on the first day
of each month. The workflow for each registered business:

1. Verify the business is still actively operating
2. Send a membership payment reminder email in the merchant's preferred language
3. Track payment status — if unpaid, escalate in subsequent months
4. After three consecutive months without payment, automatically suspend the
   business listing from the public directory
5. Include a reactivation path in the suspension email so merchants with
   legitimate billing delays can restore their listing without contacting support

The full cycle runs without any admin intervention. Suspension and reactivation
are handled entirely by the system.

### Phase 4 — Multi-Language Support and Performance

Full FR/NL/EN multi-language support was implemented across the public-facing
site and the admin CMS, allowing both the association and individual merchants
to manage content in their preferred language.

Performance optimisation was applied across the full stack — query optimisation,
eager loading, route and response caching, image optimisation at upload, and
Vue.js lazy loading on high-traffic listing routes — to support the current
10,000+ monthly visitors and a directory capacity of 1,000+ businesses.

## Outcomes

- All 250+ merchant records migrated from WordPress with zero data loss.
- Platform sustains 10,000+ monthly visitors with consistent uptime since launch.
- Membership billing engine operates fully automatically — no monthly admin
  overhead for the association.
- System capacity architected for 1,000 businesses, four times the current
  directory size.
- Full tri-language support live across all public and CMS interfaces.
- Real-time event publishing live — community events appear on the homepage
  feed immediately upon approval.
