The legacy system had no formal API and inconsistent data formatting accumulated over many years. The solution was a dedicated migration pipeline that extracted raw data from the legacy database, applied a staged transformation and validation process, and loaded records into the new schema only after each record passed a full set of integrity checks. A reconciliation report was generated after every run, comparing record counts and key field values between source and destination. The pipeline was run multiple times in a staging environment with the client verifying sample records at each stage before the production cut-over was approved."

A rollback path was kept available until the reconciliation report confirmed full data integrity on the new system."

## Overview

Promsoc Brussel is a Brussels-based student association managing memberships,
course registrations, and community communications across a multilingual
membership base. The existing platform had become unreliable — unable to handle
peak enrolment traffic, inaccessible on mobile, and locked to a single language.
I was brought in to deliver a complete rebuild: new architecture, new interface,
multilingual support, and a zero-loss migration of all existing records — nearly
10,000 users and their full registration histories — into the new system.
